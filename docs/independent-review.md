2. **Sub-objective A — role definition sync.** `docs/roles.md` > "Doc Engineer" audit checklist needs a new item 9 that describes exactly how to invoke the guardian and how to interpret its three exit codes (0/1/2), and the output template needs a slot for the violation report on FAIL. 4. **Sub-objective C — bounded recovery.** When the loop bound is hit, there must be a deterministic, non-improvised recovery path — not "Lead figures it out". The Wave 3 IR left a forward note about this. The path must preserve WIP (push the branch), document the blocked state (plan.md entry), inform the user, and exit cleanly without creating or merging a PR.

One important non-objective for Wave 4: it is NOT expected to translate any new files, introduce any new detector rules, or modify the guardian script itself. iSparto positions itself as an AI Agent Team workflow framework that turns Claude Code from a single agent into a development team with clearly defined roles. Core Features include Agent Team role separation, Wave-based parallel development, 8 slash commands, cross-session state recovery via plan.md, cross-model quality gate, automatic documentation sync, Process Observer compliance oversight (3-layer security defense), snapshot/restore, session log, and one-line install. The competitive differentiation is explicit: other AI coding tools have the user iterating with a single agent; iSparto turns that single agent into a team with clear role separation; the user only talks to the Lead, and the Lead coordinates the entire team.
